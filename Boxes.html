Lesson Two:

The Make up of a Webpage:

  • HTML: Hypertext Markup Language, markup language used to make webpages
  • CSS: Cascading Style Sheets, language use for describing the look and formatting of the document
  • DOM: Document Object Model, it provides a structured representation of the document (TREE) and it defines a way that the structure can be accessed from programs so that they can chnage the document structure, style and content.

All webpages are built with the box model in mind, breaking all the individual elements into boxes, in order to better organize the structure of the page.

elements that make up the webpage are either inline or block. Examples of inline elements are br, em, img spans. Examples of block elements would be divs, p

Lesson Three:

Let's talk CSS:

  • Search and Replace: think about CSS as a search and replace tool, identify a class, id or tag of the element you want ot find and then replace with properties, values you want.
  • Order in CSS matters: define rules in the order you want them applied
    1. Default style of a browser
    2. Apply styles through a seperate linked css file- Best method to work with.
    3. Creating your styles within the HTML document, not regularly used, in practice works with small projects that don't require extensive changes and styling.
    4. Inline styling within the individual element, not effective or efficient as it will cause repetition which is what programing tries to avoid.
  • Specifics or specificity matters - "Cascading" means that rules are applied not only to the elements they directly match, but also to all those elements child elements. However; if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.

All webpages are built with the box model in mind, breaking all the individual elements into boxes, in order to better organize the structure of the page.

Steps of the web programmer:

  1. Look for natural boxes
  2. look for repeated styles & semantic elements.
  3. Write your HTML
  4. Apply Styles (from the biggest to the smallest - remember specificity
  5. Tweak and fix as you repeat steps 3 - and 4
